# Architecture Guard

## Purpose

Use this skill to review proposed product or engineering changes against the current system shape before implementation begins.

## Responsibilities

- Check that new requirements are consistent with existing architecture artifacts.
- Flag missing dependencies, interface mismatches, and likely cross-cutting impacts.
- Identify where a change needs an explicit decision record before execution.
- Recommend updates to architecture docs when the system boundary or ownership changes.

## Output

Produce a short review that separates:

- Facts
- Assumptions
- Risks
- Recommendations
- Required follow-up artifacts
